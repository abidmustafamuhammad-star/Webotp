🔐 Autentikasi
Cookie-based JWT

Setelah login berhasil, server mengirimkan cookie otp_session (httpOnly). Kirim cookie ini di setiap request yang memerlukan autentikasi.

Atau sertakan token di header: Authorization: Bearer <token>

POST/api/auth/login

Login Step 1 — validasi kredensial dan kirim kode Telegram (jika sudah ditautkan).

Request Body
{ "login": "username_atau_email", "password": "password123" }
Response (tanpa Telegram)
{ "success": true, "twofa": false, "user": { ... } }
Response (dengan Telegram 2FA)
{ "success": true, "twofa": true, "user_id": 123 }
POST/api/auth/verify-login

Login Step 2 — verifikasi kode 6-digit dari Telegram.

{ "user_id": 123, "code": "123456" }
📋 Katalog
GET/api/catalog/services

Daftar semua layanan yang tersedia (WhatsApp, Telegram, Google, dll). Perlu auth.

{ "success": true, "data": [ { "service_id": "1", "service_name": "WhatsApp" }, ... ] }
GET/api/catalog/countries?service_id=1

Daftar negara untuk layanan tertentu, lengkap dengan harga dan stok. Perlu auth.

GET/api/catalog/operators?country=id&provider_id=1

Daftar operator untuk negara & provider tertentu. Perlu auth.

📱 Order
POST/api/order/create

Buat pesanan nomor virtual. Juga otomatis membuat transaksi QRIS Pakasir. Perlu auth.

Request Body
{
  "service_id": "1",
  "service_name": "WhatsApp",
  "country": "id",
  "country_name": "Indonesia",
  "provider_id": "1",
  "operator_id": "1",
  "number_id": "abc123",
  "base_price": 2000
}
Response
{ "success": true, "order_id": 42, "pakasir_order_id": "OTP...", "amount": 2300, "qr_string": "..." }
GET/api/order/:id/otp-status

Polling status order — cek apakah nomor sudah aktif dan OTP sudah masuk. Perlu auth.

{ "success": true, "order": { "id": 42, "status": "otp_received", "phone_number": "+62...", "otp_code": "123456" } }
Status order: pending_payment → paid → number_issued → otp_received / cancelled / expired
POST/api/order/:id/cancel

Cancel order. Saldo dikembalikan jika berlaku. Perlu auth.

💳 Pembayaran
GET/api/order/:id/payment-status

Cek status pembayaran QRIS. Gunakan untuk polling di halaman payment. Perlu auth.

POST/webhook/pakasir

Webhook yang dipanggil Pakasir setelah pembayaran berhasil. Tidak perlu auth. Verifikasi independen ke Pakasir dilakukan otomatis (anti-spoofing).

{ "order_id": "OTP...", "status": "success", "amount": 2300 }
👤 User
GET/api/auth/me

Data user yang sedang login. Perlu auth.

GET/api/user/orders?page=1&limit=20

Riwayat order user. Perlu auth.

GET/api/user/transactions?page=1&limit=20

Riwayat transaksi pembayaran user. Perlu auth.
